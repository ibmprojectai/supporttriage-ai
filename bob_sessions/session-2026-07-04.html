24. Fix SyntaxError — Escaped Quotes in f-strings (Python 3.14) Done
⚠️ Streamlit Cloud (Python 3.14.6) raised SyntaxError: unexpected character after line continuation character at ui/review_app.py:326. Python 3.14 enforces stricter f-string parsing — backslash escapes (\") inside f-string expressions are a syntax error.
  • Root cause: two f-strings used f"...{dict.get(x, \"fallback\")}..." — valid in Python ≤ 3.11, rejected in 3.14
  • Fix: replaced both expressions with "...{val}...".format(val=dict.get(x, "#fallback")) — no backslash needed
  • Lines fixed: ui/review_app.py:326 (inbox channel summary) and :526 (outage radar channels string)
  • commit 0046af4
