10. Fix extract.py — proper fix with StrOutputParser Done
  • Why attempt 1 wasn't enough: the hasattr check could still be bypassed depending on execution order; crash could recur with other LangChain response types.
  • Proper fix: chain = _PROMPT | llm | StrOutputParser() — official LangChain way to guarantee a plain str from any backend. Runs before our code touches the response.
  • Removed the 8-line hasattr workaround; replaced with 1 import + 1 chain change.
  • pipeline/extract.py — commit e045c91
11. Fix extract.py — harden _repair_json for Python 3.14 + uvloop Done
  • Crash: Streamlit Cloud (Python 3.14 + uvloop) still hit TypeError inside _repair_json at re.sub() — even with StrOutputParser in the chain.
  • Root cause: StrOutputParser on langchain-openai==1.3.3 + langchain-core==1.4.8 does not unwrap AIMessage when called with ainvoke() under uvloop — it passes it through unchanged.
  • Fix (attempt 3): changed _repair_json(raw: str) → _repair_json(raw: object), coercing to str as the first step. Still crashed because Streamlit Cloud was caching old code and serving stale versions of extract.py.
  • pipeline/extract.py — commit f771b86
12. Fix extract.py — final fix: drop chain, call llm.ainvoke() directly Done
  • Root cause (confirmed): StrOutputParser on langchain-openai 1.3.3 genuinely does not convert AIMessage → str with ainvoke(). All chain-based approaches fail on this version.
  • Final fix: dropped StrOutputParser and the LangChain chain entirely. Now calls await llm.ainvoke(prompt_text) directly on the LLM object, then immediately coerces the result with _to_str() — a dedicated helper that handles str, AIMessage, and anything else via .content or str().
  • _repair_json is now a pure string function — it only receives a guaranteed str from _to_str().
  • Falls back to asyncio.to_thread(llm.invoke, ...) if ainvoke is unavailable.
  • pipeline/extract.py — 24 insertions, 13 deletions — commit 3f5eb9b
13. Fix extract.py — definitive rewrite: remove LangChain entirely, use get_model() directly Done
  • Root cause (final): even llm.ainvoke() and _to_str() remained fragile because get_llm() returns a ChatOpenAI object — all LangChain types are unreliable on Streamlit Cloud (Python 3.14 + uvloop). The clean fix is to eliminate LangChain from the hot path entirely.
  • Definitive fix: extract.py now imports get_model() (not get_llm()). get_model() returns _OpenRouterModel on Streamlit Cloud — its generate_text() method calls the OpenRouter REST API via urllib.request and always returns a plain str. No LangChain types, no wrapping, no ambiguity.
  • Pattern is now identical to classify.py, summarize.py, and draft.py — all four stages use await asyncio.to_thread(model.generate_text, prompt).
  • Removed all LangChain imports from extract.py: no get_llm, no PromptTemplate, no StrOutputParser, no _to_str().
  • _repair_json() receives a guaranteed str — its type annotation is restored to raw: str.
  • pipeline/extract.py — commit 24ca9e4 (session log update: 93047e5)
14. Root cause found & definitive fix: retire get_llm(), harden extract.py coercion Done
⚠️ Streamlit Cloud was NOT running a stale extract.py — it was running the correct 24ca9e4 version. But something still called get_llm() which returned a ChatOpenAI object emitting AIMessage.
  • True root cause: get_llm() still existed in watsonx_config.py and returned ChatOpenAI. Streamlit Cloud's module cache or some other import path was calling it, causing AIMessage to surface even in code that called get_model(). The telltale sign was the log line [llm_config] Using OpenRouter LangChain backend — that print only existed inside get_llm().
  • Fix 1 — watsonx_config.py: get_llm() is now a 1-line shim that calls get_model(). No ChatOpenAI, no LangChain, no AIMessage possible from any caller regardless of which version of any file is cached.
  • Fix 2 — extract.py _repair_json(): Changed signature to raw: object; first thing it does is if hasattr(raw, "content"): raw = raw.content then raw = str(raw). Cannot crash on any input type.
  • Fix 3 — extract.py extract(): Added same two-line coercion immediately after asyncio.to_thread() returns — belt-and-suspenders before _repair_json is even called.
  • watsonx_config.py + pipeline/extract.py — commit e4e1659
